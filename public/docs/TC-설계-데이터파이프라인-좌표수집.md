[경로 처리 프로세스]

QField 측정 (클라우드 전송)
↓
gps_point 저장 (측정자가 수집한 원본 좌표)
↓
최적화 trigger 자동 실행
│
├─ 1. 노이즈 감지
│ - 정확도 > 10m: 점 제외
│ - 불가능한 점프 (단시간 내 30m 초과): 노이즈
│
├─ 2. 트랙 자동 분리
│ - 점프 발생 시 라인 분리 (직선 연결 방지)
│ - 점은 모두 보존
│
├─ 3. 단순화 (Douglas-Peucker)
│ - tolerance = 5m
│ - 직진 점 제거, 커브 보존
│
└─ 4. 평탄화 (Chaikin) - iterations = 3 - 부드러운 곡선화
↓
gps_track_optimized (자동 가공된 수집 경로)
↓
관리자에서 최적화된 수집경로 불러오기
├─ 배경: gps_map (회색, 반투명) - 기존 확정된 길
└─ 편집 대상: 새 트랙 (파랑) - 지금 검토 중
↓
노드, 세부사항 편집 및 저장
↓
gps_path (편집이 완료된 수집경로)
↓
gps_map (통합)

[설계 결정사항]

1. 자동화 범위
   - 자동: 측정 데이터 가공 (gps_point → gps_track_optimized)
   - 수동: 모든 판단/통합 (gps_track_optimized → gps_path → gps_map)

2. 자동화 제한 이유
   - 사유원의 세분화된 길 구조
   - GPS 정확도(5m)와 길 간격이 비슷
   - 자동 통합 시 데이터 손실 위험
   - 보수적 접근으로 안전성 우선

3. 데이터 보존
   - 모든 측정 점 gps_point에 보존
   - 노이즈로 분류된 점도 별도 처리 (자동 삭제 X)
   - 검토 후 결정

4. 처리 단위
   - 트랙 단위로 작업
   - 1번 측정 = 1개 작업 단위
   - 트랙별 개별 검토/확정

5. 핵심 개념
   - gps_path = 가지 (관리자가 확정한 개별 경로)
   - gps_map = 나무 (가지가 통합된 전체 라우팅 그래프)
